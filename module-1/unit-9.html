What is recursion?

What is recursion generally? Recursion occurs when a thing is defined in terms of itself. What is recursion used for in computer science? Recursion is an algorithm technique. Recursive algorithms break down a problem into smaller pieces, then call itself to solve the smaller pieces. Often, in the process of solving the smaller piece, the method will solve a yet smaller piece of the problem, and so on, until it reaches a "base case". A base condition occurs when the problem has been broken down to a size where it can be solved with out further breaking down. Every task performed by a recursive function is a subtask of its calling function (unless its the root). As defined above, recursion occurs when a thing is defined in terms of itself, as such we can identify an algorithm as recursive if an algorithm calls itself. Problems that can be solved recursively can generally be solved by iteration also.

Recursion vs iterative approaches

Iterative algorithms

Iteration algorithms repeat a set of instructions until a condition is met. Iterative approaches will often utilize the for and while control structures. If an iterative algorithm does not meet its exit condition, it will continue on for ever.

Recursive algorithms

Recursion is for solving problems that can be broken down into smaller, subproblems. Recursive approaches will that call selves. If a recursive function does not hit a base case, a stack overflow exception will occur when the stack memory is full.

Recursive call stack

What happens when a function is called?

Space is allocated on the stack for the functions arguments and local variables before control is passed to the function and the function runs. Once the function has finished running the result is copied the calling function to which control is returned.

int fib (int N) {
	int k1, k2, k3;
	k1 = k2 = k3 = 1;
	for (int j = 3; j <= N; j++) {
		k3 = k1 + k2;
		k1 = k2;
		k2 = k3;
	}
	return k3;
}

// recursive version
int fib (int N) {
	if ((N == 1) || (N == 2)) return 1;
	else return (fib (N-1) + fib (N-2));
}

Take away:

Solving a problem iteratively requires the explicit creation of a stack, making dynamic datasets harder to accommodate. Recursive is especially good for working on things that have many possible branches and are too complex for an iterative approach.

Benefits:

Recursion allows automatic allocation of additional objects at each function call. The iterative alternative requires the programmer to provide a mechanism for storing the required state. Recursion can be more intuitive when the approach mimics the problem, for example with Tree structures.

Disadvantage.

Recursion can be harder to debug, "down the rabbit hole". Recursion does not have a fixed memory size, whereas iterative approaches do. Recursion can be slower due to more time required allocating memory on the stack. Similarity: Both require an exit case, and if the exit case it has not reached the algorithm can continue indefinitely. Differences: If the exit case is not met for an iterative algorithm, the CPU will only waste "clock cycles", where as for a recursive algorithm memory and CPU cycles will be wasted. Recursive algorithms should be applied when the problem is itself "recursive" as the "automatic stack" can help simplify the algorithm, by allowing the user to focus on the problem one slice at a time. Recursive algorithms can be dangerous if the memory is constrained and performance should be considered if the CPU is slow.
