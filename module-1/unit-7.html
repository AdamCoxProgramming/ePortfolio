Unit 7: Data Abstraction

Compare and contrast: Linear and non-linear data structures and stacks and Queues

Linear and non-linear data structures

Linear

In a linear data structure, data is arranged sequentially (Linearly). Common Linear data structures are Arrays and (Linked)Lists. An array stores items of only one type, and stores than in sequential memory locations. A list is a linear data structure because its contents are available in a linear order to the user, its storage however in non-consecutive. Linear data structures are 1 Dimensional, and as such the term linear (arranged in or extending along a straight or nearly straight line) is suitable.

Non-linear data structures

Non-linear data structures are not arranged sequentially and are not 1 dimensional. Typical Non-linear data structures are trees and Graphs. Benefits of linear data structure Can traverse every element in a single run (linear search time). Aligns with the way memory is stored on a computer. Arrays provide efficient storage as they do not require item wrappings. Lists provide efficient insert speeds. Trees provide efficient sorted storage of data. Graphs facilitate finding the shortest distance between items.

Stack and Queue

Stacks and Queues are both linear structure.

Stacks

Stacks are Li-fo, where the last item in the fist item out. Stacks are used for the call stack where the instruction order is stored. Stacks require storing a pointer for the last element only.

Queue

Queues are Fi-fo, where the first item in is the first item out. Stores can be used to store a first come first served purchase record. Queues require storing the pointers for both the start and end elements.
