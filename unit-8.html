Unit 8: Algorithms and Basic Programming Concepts

During this unit I studied a structured framework for solving a problem applying data types, widely called, an algorithm.

Hand written algorithm:

Suppose you are sorting a set of cards. The right hand holds several cards which are sequentially sorted by numbers arranged from right to left, say for example, five cards with 10, 6, 5, 3, 2 arranged from right to left. Your left hand is picking up the top card from a pile of card deck. Now you insert the card that you have picked from the pile of deck into the right position within the right-hand cards, in a sorted order from right to left. Write the instructions in a step by step manner. Note down even a minor step that you have taken to insert the left-hand card into a correct position within the set of right-hand cards so that the right-hand card set remains sorted from right to left.

The steps for a binary search are:

Check the middle value and compare to the insert value, if the middle value is less, insert to the right, if the middle value is greaterinsert to the right. Then considering the range between the middle value and left or right end of the list, try the insert again. Keep doing this until we find the search value. For our insertion sort we will change this slighty and instead find the index of the first item that is greater than or equal to the insert value. We can then insert at the index before this index.
